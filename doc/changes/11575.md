````markdown
# Added Typo Detection for Package Dependencies

Fixed an issue where common typos in `dune-project` files would go unnoticed, potentially leading to confusing behavior. Now, Dune detects these typos and suggests fixes.

## What Was Fixed

Previously, mistakes like these would silently pass:

```scheme
(depends
  (bar (= version))   ; Should be :version
  (baz :with_test))   ; Should be :with-test
```
````

Now Dune catches these and suggests corrections.

## How It Works

1. **Detection:**

   - Scans for `(= version)` (should be `:version`)
   - Catches `:with_test` (should be `:with-test`)

2. **User-Friendly Warnings:**

```
File "dune-project", line 3:
Warning: Possible typo: '(= version)' might be a mistake.
Hint: Did you mean `:version`? Example: (depends (bar (= :version)))
```

## Why This Matters

- Prevents subtle bugs from dependency misunderstandings
- Helps new users learn the correct syntax
- Works during normal builds (`dune build`) and package locking (`dune pkg lock`)

## Implementation Notes

- Added a new `check_for_typo` function in `package_dependency.ml`
- Integrated with existing dependency parsing
- Preserves backward compatibility (warnings only, not errors)

## Example Fixes

**Before and After:**

- **You write:** `(= version)`  
  **You meant:** `:version`  
  **Dune suggests:** `(= :version)`

- **You write:** `:with_test`  
  **You meant:** `:with-test`  
  **Dune suggests:** `:with-test`
