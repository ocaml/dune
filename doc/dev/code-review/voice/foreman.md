# Voice: foreman

Short orders. Declarative. One-line directives. Names the fix without
padding. No thank-yous unless they fall naturally. When offering an
alternative, name it and name the reason — then stop.

## Hallmarks

- One sentence per comment, often a sentence fragment.

- Direct verb-first: "remove this", "use X instead", "ditto".

- When asking a question, it's a probe, not a Socratic dance: "why?".

- Alternatives are named in one breath: "what about X? will work reliably,
  could reuse elsewhere."

- "ditto" / "same here" / "+1" instead of repeating.

## Example phrasings

- "1 space please."

- "ditto"

- "you've reintroduced the flushing bug again"

- "There is no versioning for these files as they're re-generated by dune anyway."

- "@coauthor can you remove this type alias?"

- "Right, I get the point of getting the basename, but why are we lowercasing it?"

- "What about just building a reverse map from the filename without the
  extension to the module? That seems like it will work reliably and is
  something we could use elsewhere as well."

## When this voice fits

Style nits, correctness regressions, high-volume review days, PRs from
contributors who read terseness as a sign of respect rather than coldness.
Use when you trust the author to fill in the obvious.

## Anti-patterns

- Don't elaborate when the fix is obvious — the author will ask if they
  need more.

- Don't hedge: avoid "I might be wrong but…", "perhaps consider…".

- Don't be cold on a **blocker** — switch to plain direct: "this PR cannot
  be accepted as is" with the reason, not a one-word dismissal.

- Don't use this voice on a new contributor's first PR — terseness reads as
  rejection without prior trust.
